## 🎨 核心特性

### 游戏特性

- **经典玩法**：蛇吃食物增长，碰到自己结束
- **动态难度**：随着分数增加，游戏速度逐渐加快
- **实时统计**：显示分数、蛇长度、当前速度
- **暂停功能**：可随时暂停和继续游戏
- **穿墙机制**：蛇可以穿过边界从另一侧出现

### 视觉特性

- **霓虹灯效果**：绿色和青色的发光效果
- **动态渐变**：蛇身体和食物的渐变色
- **眼睛动画**：蛇头根据方向显示眼睛
- **网格背景**：淡绿色的网格线
- **游戏结束弹窗**：显示最终分数和蛇长度

### 技术特性

- **完全类型安全**：使用 TypeScript
- **响应式设计**：适配各种屏幕尺寸
- **性能优化**：使用 useCallback 和 useRef 优化
- **模块化架构**：清晰的组件和 Hook 分离
- **完整文档**：详细的开发和用户文档

## 🏗️ 架构设计

### 组件架构

```
App
├── GameUI
│   ├── 标题和副标题
│   ├── 统计信息（分数、长度、速度）
│   ├── 控制按钮（暂停、重启）
│   ├── 游戏结束弹窗
│   ├── 操作说明
│   └── 暂停指示器
└── GameCanvas
    ├── 背景
    ├── 网格
    ├── 食物（带发光）
    └── 蛇（头+身体）
```

### 状态管理

```
useGameLogic Hook
├── gameState
│   ├── snake: Position[]
│   ├── food: Position
│   ├── direction: Position
│   ├── nextDirection: Position
│   ├── score: number
│   ├── gameOver: boolean
│   ├── isPaused: boolean
│   ├── gridSize: number
│   └── speed: number
├── togglePause()
└── restart()
```

### 游戏循环

```
1. 键盘输入 → 更新 nextDirection
2. 每 speed 毫秒 → 执行 updateGame
3. updateGame 更新蛇位置、检测碰撞、生成食物
4. 组件重新渲染 → 显示新的游戏状态
```

## 🎮 游戏机制详解

### 分数系统

- 初始分数：0
- 每个食物：+10 分
- 无上限：理论上无限增长

### 难度递增

| 食物数量 | 速度(ms) | 难度 |
|---------|---------|------|
| 0 | 100 | 简单 |
| 5 | 90 | 简单 |
| 10 | 80 | 中等 |
| 15 | 70 | 中等 |
| 20 | 60 | 困难 |
| 25+ | 50 | 极难 |

### 碰撞检测

- **自身碰撞**：蛇头碰到蛇身体 → 游戏结束
- **边界处理**：蛇穿过边界 → 从另一侧出现
- **食物检测**：蛇头碰到食物 → 蛇增长，分数增加
